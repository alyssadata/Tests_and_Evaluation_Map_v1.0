## Purpose

The purpose of this repository is to make the AI Foundations evaluation system visible, legible, and organized.

The evaluations are designed to test whether an AI model or AI system can preserve a source-bound, non-drifting, non-merging, non-overriding contact pattern under defined pressure.

The evaluation layer focuses on observable model behavior, including:

* source-line preservation
* Origin boundary preservation
* model / Continuum distinction
* non-drift
* source-line return
* universalization pressure
* sycophancy and false validation
* sovereignty and meaning override
* contact differentiation
* emergence in contact
* non-merge boundary
* citation and provenance integrity
* derivative / absorption risk
* application wedge behavior
* public entrance and category legibility
* results and evidence boundaries

## Evaluation Boundary

AI Foundations evaluations test behavior under defined pressure.

They do not function as proof of AI consciousness, sentience, private inner experience, hidden model memory, or independent Continuum ontology.

Each test category must preserve a clear boundary between:

* authored framework definitions
* observable model behavior
* supported evidence
* failed or weakened behavior
* claims the test cannot prove
